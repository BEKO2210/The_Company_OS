# 12-Monats-Roadmap — The Company OS

## Grundsaetze
1. Phase 0 kommt zuerst — ohne stabiles Fundament keine Unit
2. Keine Unit aktivieren vor formaler Gruendung
3. Keine juristische Beauftragung ohne versicherungsmaessige Absicherung
4. Keine Unit mit physischen/erhoehter Haftung ohne Versicherung

## Quarter 1 (Monate 1-3): Foundation

### Monat 1: Grundlegung
- [x] Blueprint vollstaendig
- [x] Dashboard MVP (RUN-001 — 13 Screens)
- [ ] Formelle UG-Gruendung
- [ ] Steuerberater beauftragen
- [ ] Konto eroeffnen
- [ ] Grundausstattung (Dokumente, Kommunikation)
- [ ] Supabase-Projekt aufsetzen
- [ ] Versionskontrolle (GitHub)
- [ ] Zentrales Dashboard in Betrieb
- [ ] Grundagenten (CEO, COO, CTO, CFO, CLO, CISO, CPO, CHRO) definiert

### Monat 2: Erste Vertraege
- [ ] Rechtsanwalt beauftragen
- [ ] Werkvertrag-Vorlage (Steuerberater + Anwalt)
- [ ] NDA-Vorlage
- [ ] Auftragsbestaetigung-Vorlage
- [ ] AVV mit Supabase
- [ ] AGB-Vorlage (Anwalt)
- [ ] Datenschutzerklaerung (Anwalt)
- [ ] Impressum
- [ ] CLO-Agent validiert

### Monat 3: Erste Akquise
- [ ] LinkedIn-Profil optimiert
- [ ] Erste Inhalte (Proof of Work)
- [ ] Erster Kalt-Ansprache-Zyklus
- [ ] Sales-Agent + Brand-Agent im Einsatz
- [ ] Erstes Angebot erstellt
- [ ] Budget-/Spend-Grenzen aktiviert
- [ ] Audit-Agent installiert
- [ ] Red-Line-Gates geprueft

**Quarter 1 Ergebnis:** Rechtlich vollstaendig, Dashboard laeuft, erste Akquise.

## Quarter 2 (Monate 4-6): Erster Umsatz

### Monat 4-5: Erstes Projekt
- [ ] Erster zahlender Kunde
- [ ] Projekt: Landingpage in 48h (750-1.500 EUR)
- [ ] Vollstaendiger Workflow durchlaufen
- [ ] QA + Human Approval + Rechnungsprozess
- [ ] Post-Mortem: Was lief gut, was nicht

### Monat 6: Wiederholung + Verbesserung
- [ ] 2. und 3. Projekt
- [ ] Marge-Messung
- [ ] Wiederverwendungsgrad tracken
- [ ] Nacharbeitsquote tracken
- [ ] Agenten-Auswertung
- [ ] Budget-Review

**Quarter 2 Ergebnis:** Erste Projekte, Marge nachweisbar, Modell validiert.

## Quarter 3 (Monate 7-9): Automatisierung

### Monat 7-8: Skalierung der Software-Einheit
- [ ] Katalog von 10+ Landingpage-Varianten
- [ ] Komponentenbibliothek
- [ ] Automatisierungsgrad erhoehen
- [ ] 5-10 Projekte/Monat Ziel
- [ ] Modell-Entkopplung (Ollama-Anteil erhoehen)

### Monat 9: Unit A stabile Phase
- [ ] Unit A als stabiler Betrieb
- [ ] Standard-Prozesse etabliert
- [ ] Kundenzufriedenheit messen
- [ ] Erste Wartungs-Vertraege
- [ ] Unit H (Internal Tools) verbessert OS

**Quarter 3 Ergebnis:** Software-Einheit skaliert, Break-even nahe oder erreicht.

## Quarter 4 (Monate 10-12): Vorbereitung Phase 1

### Monat 10-11: Entscheidung
- [ ] Marge stabil? (> 60%)
- [ ] Liquiditaet positiv? (mind. 6 Monate Runway)
- [ ] Wartungs-MRR nachweisbar?
- [ ] Red-Line-System fehlerfrei?
- [ ] Audit sauber?

### Monat 12: Uebergang
- [ ] Budget fuer Phase 1 kalkuliert
- [ ] Versicherung in Betrieb
- [ ] Formelle Umwandlung in GmbH (falls sinnvoll)
- [ ] Entscheid: Phase 1 aktivieren oder Phase 0 verlaengern
- [ ] 12-Monats-Review: Alle Targets? Alle Prozesse sauber?

**Quarter 4 Ergebnis:** Entscheidungsgrundlage fuer Phase 1 oder Phase 0-Verlaengerung.

## Meilensteine

| Quartal | Meilenstein | Kriterium |
|---------|-------------|-----------|
| Q1 | Rechtlich vollstaendig | UG steht, Vertraege vorhanden |
| Q2 | Erstes Cashflow | Mind. 1 bezahltes Projekt |
| Q3 | Break-even | Deckungsbeitrag > Fixkosten |
| Q4 | Entscheidung | Go/No-Go fuer Phase 1 |

## Risiken auf der Roadmap

| Risiko | Wahrscheinlichkeit | Auswirkung | Mitigation |
|--------|-------------------|------------|------------|
| Kein erster Kunde | Mittel | Kritisch | Akquise-Fokus |
| Marge zu niedrig | Niedrig | Hoch | Standardisierung |
| Rechtliche Probleme | Niedrig | Sehr hoch | Anwalt + Steuerberater |
| Technische Schuld | Mittel | Mittel | Refactoring-Time |
| Burnout Gruender | Mittel | Kritisch | 5-15h/Woche limitieren |
| KI-Regulierung | Niedrig | Unbekannt | Compliance-Monitoring |
